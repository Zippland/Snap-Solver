# Snap-Solver 视觉设计提示词（Design Brief）

> 用途：将本文档整体粘贴给 UI 设计工具（或设计师），产出一套完整的视觉方案。
> 工程侧已将界面结构、交互流程全部定稿，**只需要视觉设计**：配色、字体层级、质感、关键屏 mockup。

---

## 1. 产品是什么

**Snap-Solver：局域网内的远程截屏 AI 解题工具。**

程序运行在电脑上，学生用手机浏览器连入：手机上点一下按钮 → 电脑截取它自己的整个屏幕并回传手机 → 学生在手机上框选出题目区域 → 交给自选的 AI 大模型 → 解题答案流式回传到手机，含可折叠的"思考过程"和 Markdown 格式的最终解答。

**使用情境（设计需要体感的部分）**：
- 主要在**手机竖屏**上单手操作，屏幕约 375×812；
- 高频动作是一条流水线：**截屏 → 裁剪 → 解题 → 阅读长答案**，一晚上可能重复几十次；
- 常在夜间/暗环境刷题，**暗色主题是主力**，亮色主题同等重要；
- 阅读内容以**数学步骤、代码块、公式**为主，排版可读性优先级最高；
- 用户自带 AI 模型密钥（BYO-Key），会经常在 Claude/GPT/Gemini/Qwen/豆包/Kimi 之间切换模型。

**期望气质**：一台"解题仪器"——专注、精确、快，有自己的性格，**不要**泛用 AI 产品的紫色渐变，也**不要**又一个 iOS 系统风仿制品。裁剪工作台本身是黑底全屏（类似相机取景器），整条链路的气质应与之连贯。

---

## 2. 需要交付什么

1. **色彩 Token（明暗两套完整值）**：
   - 背景三层：页面底 `bg` / 卡片浮层 `bg-elevated` / 控件填充 `fill`（两级）
   - 文字四级：主文 / 次文 / 弱文 / 禁用
   - 分隔线、遮罩
   - **一个有记忆点的品牌强调色**（用于主按钮、选中态、进度）+ 它的浅底变体（12% 左右透明度的软底）
   - 语义色：成功 / 警告 / 危险，各带软底变体
2. **字体层级**（移动端 px）：大标题 / 页标题 / 正文 / 次要 / 脚注 / 说明，含字重与行高；答案正文的阅读排版（含代码块等宽字体）单独说明
3. **形状与质感**：圆角体系（小控件/卡片/大面板/胶囊）、阴影或描边策略（注意暗色下阴影失效，需要描边或提亮补偿）、间距基准
4. **关键屏 mockup（至少 4 张，明暗各出更佳）**：
   - A. 空态主屏
   - B. 答案流式屏（生成中）
   - C. 模型选择面板（ModelSheet）
   - D. 答案屏错误状态
   （有余力可补：裁剪工作台、设置抽屉、首连引导）

## 3. 界面清单与结构（结构已定稿，请勿增删元素）

### A. 空态主屏
- 顶栏：左侧产品名 + 连接状态小胶囊（已连接=绿系 / 未连接=灰系）；右侧一个设置图标按钮
- 中央空态：一个大图标 + 标题"准备好开始了吗？" + 一句说明文字
- **底部行动栏（视觉重心）**：
  - 上方一枚**模型胶囊 chip**：显示"当前模型名 · 思考档位"（如 `Claude Sonnet 5 · 深入`），可点击，缺密钥时右侧有黄色警示点
  - 下方一个**全宽大按钮"截电脑屏"**——整个产品的主动作，要有分量

### B. 裁剪工作台（黑底全屏，类似相机取景器）
- 顶栏：左"重新截图"文字按钮 / 中提示"拖动框选出题目" / 右关闭按钮
- 中央：电脑屏幕截图 + 裁剪框（第三方裁剪库，样式可定制手柄与遮罩）
- 底部工具栏：次要按钮"重置框" + 主按钮"解题"

### C. 答案流式屏
- 顶栏：左"返回" / 中状态指示（准备中→生成中→完成/出错/已停止，带一个小转圈或状态点）/ 右"停止"小胶囊按钮（仅生成中出现）
- 题目缩略条（粘性）：左侧小截图缩略 + "点击放大对照题目"
- 滚动区：
  - **思考过程区（可折叠）**：标题行"🧠 思考过程 + 动态省略号"，内容为等宽小字号弱色长文
  - **答案正文**：Markdown 渲染——标题、列表、公式步骤、代码块（代码块右上角有"复制"小按钮）
- 右下/底部中央：流式时若用户上滑，出现"↓ 最新"悬浮胶囊
- 底部行动栏同 A（按钮文案变为"截下一题"）

### D. ModelSheet（底部弹出面板，产品最核心的新交互）
从底部滑出、顶部有 grabber 短横条，两段式：
- **上段：当前模型卡**（视觉上要突出）
  - 模型名 + 一句能力描述
  - **思考档位分段控件**：快速 / 深入 / 极限（不同模型可用档位不同），下方一行档位说明小字（如"深入 · 数十秒，带推理过程"）
  - 若该模型缺密钥：出现内联警示区——"⚠ 缺 Anthropic API Key" + 密码输入框 + "如何获取 →"链接 + 保存按钮
- **下段：切换模型列表（厂商两级手风琴，可滚动）**
  - 一级是 6 行厂商（Anthropic / OpenAI / Google / 阿里通义 / 字节豆包 / Kimi），每行：密钥状态小圆点（已配=绿 / 缺=空心灰）+ 厂商名 + 右侧辅助文字（含当前模型的厂商显示模型名并高亮，其余显示"N 款"）+ 展开箭头
  - 点厂商行展开该家的 2~3 款模型（单开互斥，当前模型所在厂商默认展开），模型行：模型名 + 选中对勾
  - 点选模型即生效，无确认按钮

### E. 设置抽屉（右侧滑入，宽约 400px）
五个分组，iOS 设置式列表：
1. 解题：提示词模式下拉 + 预览 + 新建/编辑/删除小按钮；回复语言输入
2. 密钥：六家厂商的密钥行（名称 + 已设置/未设置状态标签 + 编辑）
3. 网络：代理开关 + 主机/端口；"中转 API 地址"折叠组
4. 外观：主题三段选择（浅色/深色/跟随系统）
5. 关于：版本号、GitHub 链接

### F. 首连引导（复用 ModelSheet 的弹出面板形态，两步）
- 步骤一：图标 + "它是这样工作的" + 三句机制说明 + "知道了，去配置"
- 步骤二：即 ModelSheet 的选模型+填密钥，底部"完成"按钮 + 两个步骤圆点

### G. 轻量元素
- Toast：顶部居中小胶囊（成功/错误/信息三色图标）
- 错误恢复卡（嵌在答案正文区）：⚠ 图标 + "解题失败"标题 + 原始错误信息 + 一句人话提示 + 按钮排（重新解题 / 换个模型 / 检查密钥）
- 确认对话（如删除提示词）：同弹出面板形态，标题 + 说明 + 按钮列表（危险动作红色）

## 4. 硬约束（违反即不可实现）

1. **明暗双主题**，所有颜色成对给出；暗色是主力主题，先设计暗色
2. 移动优先 375px 宽；桌面端只是同一布局限宽居中（max-width ~680px），无需单独设计
3. iPhone 刘海/灵动岛 safe-area：顶栏与底部行动栏需预留
4. 最小触控目标 44×44px
5. 纯 CSS + 系统字体栈可实现：不依赖插画、3D、异形裁切；渐变/模糊/描边/微噪点可用
6. 图标目前用 Font Awesome 6（通用图标集），不需要定制图标包，选型时按通用形状考虑
7. 答案正文可能是几千字的数学推导或代码，**长文阅读的行高、段距、代码块对比度**是设计成败的关键
8. 裁剪工作台保持黑底（取景器语义），设计其余界面时保证与它切换不突兀

## 5. Mockup 可用的真实内容样例

- 模型名：`Claude Sonnet 5`、`GPT-5.6`、`Gemini 3.1 Pro`、`Qwen3-VL Plus`、`Doubao-Seed-1.6`、`Kimi K2.6`
- 档位：快速（几秒直接给答案）/ 深入（数十秒，带推理过程）/ 极限（最久，留给最难的题）
- 思考过程样例："先判断这是一道圆锥曲线综合题。设椭圆方程为 x²/a² + y²/b² = 1，由离心率 e = √3/2 可得…"
- 答案样例：分步解答，含"**第一步：建立坐标系**"式小标题、行内公式、一个 Python 代码块
- 错误样例："Invalid API key (401)" + 提示"看起来是密钥问题，检查一下这家的 Key 是否填对"

---

## 6. 明确不要什么

- 不要紫蓝渐变 + 星星魔法棒的"泛 AI 风"
- 不要照抄 iOS 系统设置的原生蓝 (#007AFF/#0A84FF) 灰阶模板脸
- 不要大面积插画空态、吉祥物
- 不要为了风格牺牲长文阅读性（答案区永远是内容优先）
